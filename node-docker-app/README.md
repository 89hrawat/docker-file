# Node.js Multistage Docker — Production Setup

## Project Structure

```
node-docker-app/
├── src/
│   └── index.ts          ← TypeScript source
├── dist/                 ← Compiled JS (generated by tsc, not committed)
├── Dockerfile            ← Multistage build (deps → builder → production)
├── .dockerignore         ← Exclude unnecessary files from build context
├── docker-compose.yml    ← Orchestration for local/prod
├── package.json
├── tsconfig.json
└── .env.example
```

## Docker Stages Explained

| Stage        | Purpose                              | In final image? |
|--------------|--------------------------------------|-----------------|
| `deps`       | Install all npm packages             | ❌ No           |
| `builder`    | Compile TypeScript → JavaScript      | ❌ No           |
| `production` | Run compiled app with prod deps only | ✅ Yes          |

## How to Use

### 1. Build the image
```bash
docker build -t myapp:prod .
```

### 2. Run the container
```bash
docker run -p 3000:3000 --env-file .env myapp:prod
```

### 3. Using docker-compose (recommended)
```bash
# Copy env file
cp .env.example .env

# Start
docker compose up -d

# View logs
docker compose logs -f

# Stop
docker compose down
```

### 4. Build a specific stage (for debugging)
```bash
# Only build up to the builder stage
docker build --target builder -t myapp:debug .

# Run and inspect what was compiled
docker run --rm -it myapp:debug sh
ls dist/
```

### 5. Check image size
```bash
docker images | grep myapp
# production image should be ~150MB vs ~600MB for a single-stage build
```

### 6. Check container health
```bash
docker ps                          # See health status
curl http://localhost:3000/health  # Manual check
```

## Key Files

### Dockerfile — Copy Order Matters
```dockerfile
# ✅ Copy package.json FIRST (Docker caches this layer)
COPY package*.json ./
RUN npm ci

# ✅ Copy source AFTER (only invalidates cache when source changes)
COPY . .
RUN npm run build
```

### .dockerignore — Always Include
Prevents `node_modules`, `.env`, and source files from being
sent to the build context, making builds faster and safer.
